\*Item range indicator is especially valuable in data-heavy interfaces where users need to navigate through large sets of information efficiently. It may be omitted where space is limited or such precise context isn't necessary.

Note: The page navigation and unselected page buttons use the secondary variant of the button component. The selected (also known as current) page button uses the secondary variant with disabled state styles, but with onSecondary color token and bold font for the text instead.

## Options

### Numeric display

The numeric display option renders page numbers as interactive, clickable buttons in a familiar sequential format, allowing users to quickly jump to specific pages.

```jsx withPreview
export default function Example() {
  const [page, setPage] = useState(1);
  const [page2, setPage2] = useState(2);
  const [page5, setPage5] = useState(5);
  const [page14, setPage14] = useState(14);
  const [page15, setPage15] = useState(15);

  // for item range indicator
  const items = 150;
  const showTotal = true;

  const MAX_RESULTS_PER_PAGE = 10;
  const totalPages = Math.ceil((items ?? 0) / MAX_RESULTS_PER_PAGE);
  const endlessPagination = totalPages > 15;

  const variant = "numeric";

  const slug = "";

  const DisplayContainer = styled("div", {
    gap: "$025",
    display: "flex",
    flexWrap: "nowrap",
    alignItems: "center",
    padding: "8px 0px",
  });

  return (
    <Box css={{ flexDirection: "column" }}>
      <Pagination.Root
        endlessPagination={endlessPagination}
        page={page}
        setPage={setPage}
        showItems={true}
        slug={slug}
        totalPages={totalPages}
        variant={variant}
      >
        <DisplayContainer>
          <Pagination.PreviousButton />
          <Pagination.Display />
          <Pagination.NextButton />
        </DisplayContainer>
      </Pagination.Root>

      <Pagination.Root
        endlessPagination={endlessPagination}
        page={page2}
        setPage={setPage2}
        showItems={true}
        slug={slug}
        totalPages={totalPages}
        variant={variant}
      >
        <DisplayContainer>
          <Pagination.PreviousButton />
          <Pagination.Display />
          <Pagination.NextButton />
        </DisplayContainer>
      </Pagination.Root>

      <Pagination.Root
        endlessPagination={endlessPagination}
        page={page5}
        setPage={setPage5}
        showItems={true}
        slug={slug}
        totalPages={totalPages}
        variant={variant}
      >
        <DisplayContainer>
          <Pagination.PreviousButton />
          <Pagination.Display />
          <Pagination.NextButton />
        </DisplayContainer>
      </Pagination.Root>

      <Pagination.Root
        endlessPagination={endlessPagination}
        page={page14}

## Behavior

### Triggers

- When clicking the next or previous page buttons, the page numbers update and content transitions smoothly.
- When clicking on the fifth page from the first and the last pages, it’s selected state will appear as the middle page number, and the overflow buttons appear.
- The overflow button triggers the next set of sequential numbers, allowing users to jump three pages.

```jsx withPreview

## Guidance

### Endless pagination

Endless pagination, also known as infinite scroll, should be reserved for content-heavy scenarios like article feeds or large datasets, as implementing it for standard navigation can lead to poor user orientation, difficulty in reaching specific pages, and potential performance issues. Including the first page in the numeric display variant can help users return quickly to the beginning of their content journey without having to navigate through multiple steps.

```jsx withPreview isGuide="success"
export default function Example() {
  const [page, setPage] = useState(40);
  const [page2, setPage2] = useState(40);

  const items = 1053;

  const MAX_RESULTS_PER_PAGE = 10;
  const totalPages = Math.ceil((items ?? 0) / MAX_RESULTS_PER_PAGE);
  const endlessPagination = totalPages > 15;

  const variant = "numeric";

  const slug = "";

  const DisplayContainer = styled("div", {
    gap: "$025",
    display: "flex",
    flexWrap: "nowrap",
    alignItems: "center",
    padding: "8px 0",
  });

  return (
    <Box css={{ flexDirection: "column" }}>
      <Pagination.Root
        endlessPagination={endlessPagination}
        page={page}
        setPage={setPage}
        slug={slug}
        totalPages={totalPages}
        variant={variant}
      >
        <DisplayContainer>
          <Pagination.PreviousButton />
          <Pagination.Display />
          <Pagination.NextButton />
        </DisplayContainer>
      </Pagination.Root>

      <Pagination.Root
        css={{ justifyContent: "center" }}
        endlessPagination={endlessPagination}
        page={page2}
        setPage={setPage2}
        slug={slug}
        totalPages={totalPages}
        variant="compact"
      >
        <DisplayContainer>
          <Pagination.PreviousButton />
          <Pagination.Display />
          <Pagination.NextButton />
        </DisplayContainer>
      </Pagination.Root>
    </Box>
  );
}
```

### Mobile and smaller screens

Implement a responsive pagination component with a mobile-optimized fallback variant. The component should dynamically switch to a compact UI based on viewport dimensions or container constraints, ensuring consistent UX while maintaining core navigation features across all device sizes.

```jsx withPreview isGuide="success"
export default function Example() {
  const [page, setPage] = useState(1);
  const [page2, setPage2] = useState(40);

  // for item range indicator
  const items = 150;
  const showTotal = true;

  const MAX_RESULTS_PER_PAGE = 10;
  const totalPages = Math.ceil((items ?? 0) / MAX_RESULTS_PER_PAGE);
  const endlessPagination = totalPages > 15;
  const showItems = true;

  const variant = "numeric";

  const slug = "";

  const DisplayContainer = styled("div", {
    gap: "$025",
    display: "flex",
    flexWrap: "nowrap",
    alignItems: "center",
  });

  const StyledP = styled("p", {
    fontWeight: theme.fontWeights.bold,
    fontSize: theme.fontSizes["087"],
  });

  return (
    <DisplayContainer css={{ flexDirection: "column" }}>
      <StyledP>
        Uses numeric variant and shows range of items on desktop:
      </StyledP>
      <Box css={{ flexDirection: "column" }}>
        <Pagination.Root
          endlessPagination={endlessPagination}
          page={page}
          setPage={setPage}
          showItems={showItems}
          slug={slug}
          totalPages={totalPages}
          variant={variant}
        >
          <Pagination.ItemRangeIndicator showTotal={showTotal} items={items} />
          <DisplayContainer>
            <Pagination.PreviousButton />
            <Pagination.Display />
            <Pagination.NextButton />
          </DisplayContainer>
        </Pagination.Root>
        <StyledP>
          Uses compact variant and hides range of items on mobile devices or
          smaller screens:
        </StyledP>
        <div>
          <Pagination.Root
            css={{ justifyContent: "center" }}
            endlessPagination={endlessPagination}
            page={page}
            setPage={setPage}
            showItems={showItems}
            slug={slug}
            totalPages={totalPages}
            variant="compact"
          >
            <DisplayContainer>
              <Pagination.PreviousButton />
              <Pagination.Display />
              <Pagination.NextButton />
            </DisplayContainer>
          </Pagination.Root>
        </div>
      </Box>
    </DisplayContainer>
  );
}
```
